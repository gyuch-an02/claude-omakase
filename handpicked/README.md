# handpicked/

Overlay entries that adapters don't (or can't) generate well. Drop one JSON file per entry.

## When to add an entry here

- A Claude skill exists that no adapter currently scrapes (e.g. only documented in a blog post).
- An adapter produces a version of the entry, but the install metadata is wrong and you want to **override** it. Use the same `id` — the registry sorts handpicked first, so it wins.
- You've personally audited a community entry and want to mark it `verified: true`.

## When NOT to add an entry here

- The entry can be generated by extending an existing adapter — prefer that. Adapters scale; handpicked entries do not.
- You haven't actually tried the install. We do not ship `verified: true` for unaudited claims.

## Schema

See `../src/types.ts::Entry`. Required at minimum:

- `id` (kebab-case, unique)
- `name`, `description`
- `type` (`claude_code_skill` | `claude_skill`)
- `tags` (non-empty)
- `verified` (only `true` if you've audited the publisher)
- `author.name`
- `install.skill_files` (list of source URL → target relative path under `~/.claude/skills/<id>/`)
- `source.adapter: "handpicked"` (the registry sets this if you omit it)

## Example

```json
{
  "id": "git-commit-helper",
  "name": "Git Commit Helper",
  "type": "claude_code_skill",
  "description": "Drafts conventional-commit messages from staged diffs.",
  "tags": ["git", "commit", "workflow"],
  "verified": true,
  "author": { "name": "Anthropic", "url": "https://github.com/anthropics/skills" },
  "homepage": "https://github.com/anthropics/skills/tree/main/git-commit-helper",
  "license": "MIT",
  "install": {
    "command": "noop",
    "skill_files": [
      {
        "source": "https://raw.githubusercontent.com/anthropics/skills/main/git-commit-helper/SKILL.md",
        "target": "SKILL.md"
      }
    ]
  },
  "source": { "adapter": "handpicked" }
}
```
